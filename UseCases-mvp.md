# Общие бизнес-правила MVP 

1. Пользователь видит и управляет только своими тикетами.
2. Тикет обязательно принадлежит категории.
3. Категория принадлежит теме.
4. Сотрудник поддержки видит тикеты только своей категории.
5. Сотрудник может изменять только назначенные на него тикеты.
6. Администратор имеет полный доступ к структуре и тикетам.
7. Изменение статуса и добавление комментариев вызывает отправку уведомления.



# Пользователь



##  UC-1: Регистрация

| Поле              | Значение                           |
|-------------------|------------------------------------|
| Актор             | Пользователь                       |
| Описание          | Создание учетной записи            |
| Предусловие       | Пользователь не зарегистрирован    |
| Постусловие       | Создана новая учетная запись       |
| Основной сценарий | 1. Пользователь вводит данные.     |
|                   | 2. Система валидирует данные.      |
|                   | 3. Система сохраняет пользователя. |
| Альтернативы      | Email уже существует -> ошибка.    |
